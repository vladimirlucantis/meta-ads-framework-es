# Contribuir a meta-ads-framework-es

Gracias por considerar contribuir. Este proyecto está en fase inicial y toda la ayuda suma.

## Cómo contribuir

### Reportar bugs o mejoras

1. Abrí un [issue](https://github.com/vladimirlucantis/meta-ads-framework-es/issues)
2. Explicá qué esperabas que hiciera el skill y qué hizo
3. Si podés, incluí el prompt que usaste y la respuesta que obtuviste

### Proponer cambios

1. Forkeá el repo
2. Creá una rama descriptiva: `feat/nueva-seccion-umbrales` o `fix/correccion-frecuencia`
3. Hacé tus cambios
4. Abrí un PR explicando qué cambia y por qué

## Qué tipo de aportes busco

**Bienvenidos:**

- Correcciones de umbrales basadas en datos reales de campañas
- Nuevos patrones de creativos FRÍA/CALIENTE probados
- Traducciones del skill a otros idiomas (portugués, inglés)
- Ejemplos adicionales en `examples/`
- Mejoras de claridad en el SKILL.md

**Con cuidado:**

- Datos de clientes reales (siempre anonimizar, nunca publicar nombres, pixel IDs, facturación real)
- Cambios a la estructura de archivos del skill (puede romper la detección automática)

**Fuera de alcance:**

- Generadores de copy genérico sin criterio
- Integraciones con APIs pagas sin documentar el costo al usuario
- Umbrales calibrados solo para mercados fuera de LATAM (pueden ir en un fork)

## Estilo

- Español rioplatense (voseo). Si aportás en español neutro, anotalo en el PR para revisar
- Markdown con headers cortos
- Evitar anglicismos cuando hay equivalente en español. "Segmentación" antes que "targeting"
- Ejemplos con números reales (aunque sean anonimizados), no con "X pesos"

## Código de conducta

Sé amable. Asumí buena fe. Discutí ideas, no personas.

## Preguntas

Abrí un issue con el tag `question` o escribime por GitHub.
